
# Java Game Engine — Software Engineering Project
 
A desktop game built in Java as part of a university Software Engineering module, covering the full SDLC from requirements through to a tested, working implementation (B+ grade).
 
## Overview
 
This project simulates a real-world software delivery cycle rather than just a coding exercise. It includes requirements analysis, UML use-case modelling, system design, a Java Swing GUI, object persistence (save/resume), and a JUnit-tested codebase, all delivered using a Kanban-based agile workflow.
 
## Features
 
- **Save/Load System** — Object persistence allows players to save progress mid-game and resume from their last saved state.
- **GUI** — Interactive Java desktop interface for gameplay and navigation.
- **Tested Core Logic** — Key game logic and persistence components covered by JUnit unit tests.
- **Structured Design** — Built from formal use-case models and system design diagrams before implementation began.

## Software Engineering Process
 
This project followed a structured engineering lifecycle:
 
1. **Requirements Analysis** — Gathered and documented functional/non-functional requirements.
2. **Use-Case Modelling** — Mapped user interactions and system behaviour with UML use-case diagrams.
3. **System Design** — Designed class structure and persistence model ahead of coding.
4. **Implementation** — Built the game logic, GUI, and save system in Java.
5. **Testing** — Wrote and ran JUnit tests against core functionality.
6. **Delivery** — Managed using a Kanban board to track tasks and iterate incrementally.
